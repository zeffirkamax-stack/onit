## Лабораторная 4 — демонстрация балансировки нагрузки Nginx (round-robin)

Это простое приложение на Node.js, упакованное в Docker-контейнер.  
Три экземпляра приложения балансируются через Nginx по методу **round-robin**.

При обновлении одной и той же ссылки текст будет меняться:  
отображаются `Нода 1`, `Нода 2`, `Нода 3` — в зависимости от того, какой контейнер обслужил запрос.

### Структура

- `app/server.js` — веб-приложение на Express, выводит название ноды из переменной окружения `NODE_NAME`.
- `app/package.json` — зависимости Node.js.
- `Dockerfile` — образ backend-приложения.
- `nginx/nginx.conf` — конфигурация Nginx с `upstream` и round-robin по трём нодам.
- `docker-compose.yml` — поднимает три backend-контейнера (`app1`, `app2`, `app3`) и Nginx.

### Как запустить

1. Убедись, что установлен Docker и Docker Compose.
2. В корне проекта (`Lab4`) выполни команду:

   ```bash
   docker-compose up --build
   ```

3. Открой в браузере:

   - `http://localhost:8080/`

4. Обновляй страницу несколько раз (F5 / Ctrl+R) —  
   Nginx по очереди будет отправлять запросы в `app1`, `app2`, `app3`, и на странице
   ты увидишь:

   - `Нода 1`
   - `Нода 2`
   - `Нода 3`

### Остановка

В той же директории:

```bash
docker-compose down
```
